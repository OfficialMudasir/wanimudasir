Alvarnet CEP (Client Experience Portal)

Overview

The Alvarnet CEP (Centralized Engagement Platform) is a modern, scalable, and secure multi-portal system designed to manage client organizations, users, and operational insights in a centralized manner. The platform enables administrators and client users to interact with a unified system for managing organizational data, monitoring activities, and handling user access efficiently.

The solution is built with a decoupled architecture, consisting of a backend API and separate frontend portals, ensuring flexibility, maintainability, and scalability.

Core objectives

  • Provide a centralized platform for managing multiple client organizations (tenants)
  • Enable secure user management and authentication
  • Offer real-time insights and activity tracking
  • Ensure scalability and maintainability through modular architecture
  • Support future multi-tenant expansion and enterprise integrations

Architecture overview

1) Backend (CEP.API)

  • Built using ASP.NET Core (.NET 9)
  • Layered architecture: Controllers, Services, Repositories, Core & Infrastructure
  • RESTful APIs with standardized error handling using RFC7807 ProblemDetails
  • Logging + exception handling
  • Readiness/health endpoint: /ready
  • Designed for secure and scalable API communication

2) Admin Portal (CEP.UI.Admin)

  • Organization (Tenant) management
  • Admin user management
  • CEP user management
  • Dashboard insights (clients, users, incidents, etc.)
  • Activity tracking (Recent Admin Activity)

3) Client Portal (CEP.UI.Client)

  • Access to organization-specific data
  • User-level operations
  • Settings and documentation access
  • Focused on usability and performance (e.g., skeleton loaders)

Key features

  • Multi-tenant ready design: Designed for future true multi-tenant implementation with logical separation of tenant data
  • User & role management: RBAC, secure authentication and authorization flows
  • Organization management: Create/update/manage tenant configuration and metadata
  • Activity monitoring: Visibility into recent operations for auditing and monitoring
  • Robust error handling: ProblemDetails responses with traceability (traceId)
  • Improved UX: Skeleton screens, clear validation/errors, and action feedback loaders

Testing & quality assurance

  • Unit testing (backend) using xUnit, Moq, and FluentAssertions
  • Planned E2E automation using Cypress
  • Designed for CI/CD integration with Azure DevOps pipelines

DevOps & deployment

  • Hosted on Azure (Static Web Apps + API services)
  • CI/CD via Azure DevOps, environment-based configuration
  • Readiness and health checks for deployment validation

Scalability & future scope (CEP v2)

  • Full multi-tenant architecture (domain/tenant-based resolution)
  • Enhanced authentication (Entra ID + JWT hybrid approach)
  • Advanced analytics and reporting
  • Automation and monitoring improvements
  • Expanded API coverage and integrations

Conclusion

The Alvarnet CEP platform is a robust, extensible, and enterprise-ready solution designed to streamline organization and user management across multiple clients. With a strong foundation in v1 and a clear roadmap for v2, the platform is well-positioned for scalability, improved automation, and enhanced user experience.
